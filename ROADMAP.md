# Easy-Station Roadmap

> 最后更新：2026-03-23

## 项目定位

Easy-Station 是一个**目标驱动的 Agent 运维平台**，聚焦三大核心：

1. **目标驱动交互** - 围绕"部署 Agent"、"执行命令"等目标设计 UI
2. **Server-Agent 分离** - 控制面与数据面解耦
3. **Agent 插件化** - Host Agent 负责调度，Agent Plugin 负责具体能力

## 边界约束

### ✅ 在范围内

| 能力 | 说明 |
|------|------|
| Agent 部署 | 资源拉取、打包、部署、状态追踪 |
| 命令执行 | 单机/批量命令、模板化命令、执行历史 |
| 状态监控 | 心跳上报、健康检查、日志查看 |
| 环境管理 | Environment、Host、Agent 实例管理 |
| 权限控制 | RBAC、审计日志 |
| 模板机制 | AgentTemplate → AgentInstance → Binding |

### ❌ 超出范围（独立产品）

以下功能**不在** Easy-Station 范围内：

| 功能 | 原因 |
|------|------|
| AIOps / 智能运维 | AI 产品 |
| IoT / 边缘计算 | IoT 平台 |
| 容灾管理 | 容灾系统 |
| 工单系统 | ITSM 产品 |
| 报表中心 | BI 产品 |
| 工作流引擎 | 独立产品 |
| 租户门户 | 门户产品 |
| 成本核算 | 财务系统 |
| 知识库 | 文档系统 |
| 移动端 APP | 移动开发 |
| 测试平台 | QA 产品 |
| 多租户隔离 | 架构决策（Phase 3+） |
| 灰度发布 | 流量治理（Phase 3+） |
| SSO/OAuth | 企业集成（Phase 3+） |

## Issue 管理规范

### 标签体系

| 标签 | 用途 |
|------|------|
| `phase:1` | MVP 核心功能 |
| `phase:2` | 扩展能力 |
| `scope:out` | 超出范围（关闭时使用） |
| `type:feature` | 新功能 |
| `type:bug` | Bug 修复 |
| `type:docs` | 文档 |
| `type:chore` | 工程/维护 |
| `priority:high` | 高优先级 |
| `priority:medium` | 中优先级 |
| `priority:low` | 低优先级 |
| `status:todo` | 待开发 |
| `status:in-progress` | 开发中 |
| `status:blocked` | 阻塞 |
| `module:server` | 服务端模块 |
| `module:agent` | Agent 模块 |
| `module:frontend` | 前端模块 |

### 创建 Issue 前必查

1. **是否在项目边界内？** 参考"边界约束"章节
2. **是否已有类似 Issue？** 搜索避免重复
3. **是否属于 Phase 1？** 非核心功能放 Phase 2+
