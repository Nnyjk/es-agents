## 第一章：平台介绍

### 1.1 ES Agents 是什么

ES Agents（Easy-Station Agents）是一个**目标驱动的自动化运维平台**，通过 Server-Agent 架构实现分布式系统的统一管理和自动化操作。

**核心价值：**

- 🎯 **目标驱动**：声明式配置，系统自动达成目标状态
- 🔌 **插件化 Agent**：可扩展的任务执行能力
- 📊 **可视化监控**：实时状态感知与告警
- 🚀 **自动化部署**：一键部署与回滚

### 1.2 核心概念

| 概念            | 说明                                     |
| --------------- | ---------------------------------------- |
| **Server**      | 平台控制中枢，提供 API、UI、任务调度     |
| **Agent**       | 分布式执行节点，接收 Server 指令执行任务 |
| **Host**        | 被管理的主机资源，Agent 运行载体         |
| **Environment** | 环境隔离（开发/测试/生产）               |
| **Deployment**  | 应用部署配置与执行记录                   |
| **Plugin**      | Agent 插件，扩展任务执行能力             |

## 第二章：快速入门

### 2.1 系统要求

**Server 端：**

- Java 17+
- 内存：2GB+
- 磁盘：10GB+
- 数据库：PostgreSQL 14+

**Agent 端：**

- Go 1.21+（或使用预编译二进制）
- 内存：512MB+
- 网络：可访问 Server 端

### 2.2 安装部署

#### Server 端安装

```bash
# 1. 克隆仓库
git clone https://github.com/Nnyjk/es-agents.git
cd es-agents

# 2. 配置数据库
cp server/src/main/resources/application.properties.example \
   server/src/main/resources/application.properties
# 编辑配置文件，设置数据库连接

# 3. 构建并启动
cd server
./mvnw clean package -DskipTests
java -jar target/quarkus-app/quarkus-run.jar
```

#### Agent 端安装

```bash
# 1. 编译 Agent
cd agent
go build -o esa-agent

# 2. 配置 Agent
cp config.yaml.example config.yaml
# 编辑配置文件，设置 Server 地址、认证信息

# 3. 启动 Agent
./esa-agent
```

### 2.3 首次登录与配置

1. **访问平台**

   - 打开浏览器访问 `http://localhost:8080`
   - 使用默认管理员账号登录（首次启动时创建）

2. **初始配置**

   - [截图：登录页面]
   - 进入「系统设置」→「环境管理」
   - 创建第一个环境（如：Development）

3. **添加主机**

   - 进入「主机管理」
   - 点击「添加主机」
   - 填写主机信息（IP、SSH 凭证等）
   - [截图：添加主机表单]

4. **部署 Agent**
   - 进入「Agent 管理」
   - 选择目标主机
   - 点击「部署 Agent」
   - 等待 Agent 上线

### 2.4 基础使用流程

```
1. 创建环境 → 2. 添加主机 → 3. 部署 Agent → 4. 执行任务
```

## 第三章：功能模块指南

### 3.1 环境管理

**功能说明：** 管理不同环境的隔离配置（开发、测试、生产等）。

#### 创建环境

1. 进入「环境管理」页面
2. 点击「新建环境」按钮
3. 填写环境信息：
   - **名称**：如 `Development`、`Production`
   - **描述**：环境用途说明
   - **颜色标识**：用于 UI 区分
4. 点击「保存」

[截图：创建环境表单]

#### 编辑/删除环境

- **编辑**：点击环境卡片右上角「编辑」按钮
- **删除**：点击「删除」按钮（需确认无关联资源）

#### 环境列表查看

- 查看所有环境及关联的主机、部署数量
- 支持按名称搜索、按创建时间排序

---

### 3.2 主机管理

**功能说明：** 管理被管控的主机资源，包括 SSH 连接、状态监控。

#### 添加主机

1. 进入「主机管理」页面
2. 点击「添加主机」
3. 填写主机信息：

| 字段     | 说明         | 示例                                 |
| -------- | ------------ | ------------------------------------ |
| 名称     | 主机标识     | `web-server-01`                      |
| IP 地址  | 主机 IP      | `192.168.1.100`                      |
| SSH 端口 | SSH 服务端口 | `22`                                 |
| 认证方式 | 密码/密钥    | `密钥`                               |
| 用户名   | SSH 登录用户 | `root`                               |
| 私钥     | SSH 私钥内容 | `-----BEGIN RSA PRIVATE KEY-----...` |

4. 点击「测试连接」验证配置
5. 点击「保存」

[截图：添加主机表单]

#### 主机配置

- **编辑**：修改主机信息、SSH 凭证
- **标签管理**：为主机添加标签便于分组
- **关联环境**：将主机分配到指定环境

#### 主机状态监控

- **在线状态**：实时显示 Agent 连接状态
- **资源监控**：CPU、内存、磁盘使用率
- **最后心跳**：Agent 最后上报时间

[截图：主机列表状态]

---

### 3.3 Agent 管理

**功能说明：** 管理分布式 Agent 实例，包括部署、配置、监控。

#### Agent 模板

- 预配置的 Agent 部署模板
- 支持自定义插件配置
- 一键应用到多个主机

#### Agent 实例管理

1. 进入「Agent 管理」页面
2. 查看已部署的 Agent 列表
3. 操作选项：
   - **重启**：远程重启 Agent 服务
   - **升级**：升级到最新版本
   - **卸载**：从主机移除 Agent

[截图：Agent 列表]

#### Agent 状态监控

- **连接状态**：WebSocket 连接状态
- **任务执行**：当前/历史任务执行情况
- **插件状态**：已加载插件及状态
- **指标监控**：实时性能指标

---

### 3.4 部署管理

**功能说明：** 管理应用的自动化部署流程。

#### 创建部署

1. 进入「部署管理」页面
2. 点击「新建部署」
3. 配置部署信息：

| 字段     | 说明                   |
| -------- | ---------------------- |
| 部署名称 | 部署任务标识           |
| 目标环境 | 选择部署环境           |
| 目标主机 | 选择部署主机（可多选） |
| 部署策略 | 蓝绿/滚动/重建         |
| 应用配置 | 镜像/包路径、启动参数  |

4. 点击「保存并执行」或「保存」

[截图：创建部署表单]

#### 部署流水线

标准部署流程：

```
1. 预检查 → 2. 停止服务 → 3. 部署新包 → 4. 启动服务 → 5. 健康检查
```

- 支持配置自定义步骤
- 支持步骤失败回滚

#### 部署历史与回滚

- **历史记录**：查看所有部署执行记录
- **状态查看**：成功/失败/进行中
- **一键回滚**：快速回滚到历史版本

[截图：部署历史列表]

### 3.5 命令执行

**功能说明：** 远程执行 Shell 命令，支持批量执行。

2. 选择目标主机（支持多选）

#### 命令历史

#### 执行结果查看

- **实时输出**：命令执行过程中的实时日志
- **退出码**：命令执行结果状态
- **执行时间**：命令执行耗时

[截图：告警规则配置]

#### 告警通知

- **通知渠道**：支持邮件、钉钉、Webhook
- **通知模板**：自定义告警消息格式
- **静默规则**：配置免打扰时段

#### 告警历史

- 查看所有告警记录
- 告警状态：触发、确认、恢复
- 告警统计：按时间、级别、类型统计

### 3.8 消息中心

**功能说明：** 站内消息通知中心，集中管理平台通知、告警、系统消息。

  - 消息标题与内容

- 消息分布：按类型、级别统计消息数量

#### 实时推送

## 第四章：常见问题解答

### 4.1 安装问题

**Q: Server 启动失败，提示数据库连接错误**

A: 检查以下配置：

1. 确认 PostgreSQL 服务已启动
2. 检查 `application.properties` 中数据库连接配置
3. 确认数据库用户有足够权限
4. 检查防火墙是否允许数据库端口访问

**Q: Agent 无法连接 Server**

A: 排查步骤：

1. 检查 Agent 配置中 Server 地址是否正确
2. 确认 Server 端 WebSocket 服务正常（默认 8080 端口）
3. 检查网络连通性（防火墙、安全组）
4. 查看 Agent 日志定位具体错误

### 4.2 连接问题

**Q: 主机 SSH 连接测试失败**

A: 检查项：

1. 确认主机 IP 和 SSH 端口正确
2. 验证 SSH 凭证（密码或私钥）
3. 确认 SSH 服务已启动
4. 检查 Server 到主机的网络连通性

**Q: Agent 频繁掉线**

A: 可能原因：

1. 网络不稳定，检查网络连接
2. Server 负载过高，检查 Server 资源
3. Agent 配置的心跳间隔过短
4. 主机资源不足导致 Agent 异常

### 4.3 部署问题

**Q: 部署任务执行失败**

A: 排查流程：

1. 查看部署日志定位失败步骤
2. 检查目标主机磁盘空间
3. 确认应用包路径可访问
4. 验证部署脚本权限

**Q: 部署回滚失败**

A: 检查项：

1. 确认历史版本包仍存在
2. 检查回滚脚本是否正确
3. 验证目标主机状态正常

### 4.4 其他常见问题

**Q: 如何备份平台数据？**

A: 备份以下内容：

1. PostgreSQL 数据库（`pg_dump`）
2. Server 配置文件
3. Agent 配置文件

**Q: 如何升级到新版本？**

A: 升级步骤：

1. 备份当前数据和配置
2. 停止 Server 和 Agent 服务
3. 拉取新版本代码/二进制
4. 执行数据库迁移（如有）
5. 启动新版本服务

**Q: 如何添加自定义 Agent 插件？**

A: 参考《开发者指南》- Agent 插件开发章节。

### B. 配置文件示例

#### Server 配置 (application.properties)

```properties
# 数据库配置
quarkus.datasource.db-kind=postgresql
quarkus.datasource.username=esa
quarkus.datasource.password=esa-password
quarkus.datasource.jdbc.url=jdbc:postgresql://localhost:5432/es_agents

# Server 端口
quarkus.http.port=8080

# WebSocket 配置
quarkus.websocket.max-frame-size=65536
```

#### Agent 配置 (config.yaml)

```yaml
server:
  url: ws://localhost:8080/api/v1/agent/ws
  token: your-agent-token

host:
  name: agent-01
  environment: development

plugins:
  enabled:
    - shell
    - file
    - deployment
```
